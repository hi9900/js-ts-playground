# 📚 Learning Folder Guide

강의 기반으로 학습한 내용을 체계적으로 기록하기 위한 가이드입니다.  
강의별 정리, 실습 코드, 복습용 자료를 일관성 있게 관리하는 것을 목표로 합니다.

---

## 🗂️ 폴더 구조 규칙

`learning` 폴더는 **강의 단위**로 하위 폴더를 생성합니다.

```
learning/
└── [언어]-[강의명]/
    ├── README.md       # 강의 전체 요약 + 목차
    ├── note/           # 강의 섹션/파트별 학습 기록
    ├── code/           # 실습 코드
    └── exercises/      # 과제/연습 문제 풀이
```

---

## 🧱 폴더 이름 규칙 (Naming Convention)

> 강의가 여러 개 쌓였을 때도 **정렬, 검색, 구분이 쉬운** 형태 유지

### 형식

```
{language}-{course-name}
{language}-{course-name}-{platform} # 플랫폼이 필요한 경우
```

### 규칙

| 요소                 | 설명                                         |
|--------------------|--------------------------------------------|
| 언어(language)       | js, ts, node, react 등                      |
| 강의명(course-name)   | 짧고 핵심만. 2~3 단어 요약 추천                       |
| 플랫폼(platform) (선택) | inflearn, nomadcoders, fastcampus, udemy 등 |

---

## 📑 각 강의 폴더 내부 구성

| 파일/폴더        | 용도                     |
|--------------|------------------------|
| `README.md`  | 강의 전체 개요, 목차, 핵심 요약    |
| `notes/`     | 섹션/파트별 개별 노트           |
| `code/`      | 수업 실습 코드               |
| `exercises/` | 직접 푼 문제, 응용 과제, 챌린지 코드 |

---

## 🧭 추천 학습 흐름

1. 강의 시작 시
    - 폴더 생성 + `README.md`에 강의 정보 기록
2. 수강 중/직후
    - 섹션별 `notes/` 작성
3. 실습 단계
    - `code/`에 실습 정리
4. 강의 종료 후
    - `README.md`에 최종 요약 업데이트
    - 배운 내용 중 "프로젝트에 적용할 아이디어" 따로 기록

---

## 📦 확장 가능 폴더 (선택)

필요 시 아래 구조 추가 가능:

| 폴더           | 용도                 |
|--------------|--------------------|
| `resources/` | 참고자료, PDF, 링크, 이미지 |
| `quiz/`      | 복습용 테스트, 스스로 문제 출제 |
| `summary/`   | 핵심 요약만 모아둔 압축 버전   |
